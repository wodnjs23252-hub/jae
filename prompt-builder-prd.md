# 프롬프트 빌더 PRD

## 제품 개요

프롬프트 빌더(가칭)는 AI를 처음 접하는 일반인과 업무 생산성을 높이려는 직장인이 자신의 단순한 아이디어, 업무 목적, 원하는 결과 형태를 입력하면 이를 구조화된 고품질 프롬프트로 자동 변환해주는 웹 및 모바일 기반 생산성 도구이다.

이 서비스는 사용자가 프롬프트 엔지니어링 원칙을 직접 학습하지 않아도 Role, Context, Task, Constraints, Output Format 등 핵심 구성 요소를 자연스럽게 입력하도록 돕고, 최종적으로 바로 복사해 ChatGPT, Claude, Gemini 등 생성형 AI 도구에 사용할 수 있는 프롬프트를 제공한다.

핵심 가치 제안은 다음과 같다.

- 초보자도 질문에 답하는 방식만으로 실용적인 프롬프트를 만들 수 있다.
- 직장인은 보고서, 이메일, 기획서, 마케팅 문구, 코드 요청 등 반복 업무용 프롬프트를 빠르게 생성할 수 있다.
- 템플릿 기반 구조화와 실시간 미리보기를 통해 결과 품질과 사용 신뢰도를 높인다.
- 웹과 모바일에서 동일한 흐름으로 사용할 수 있는 반응형 인터페이스를 제공한다.

## 1. 목표 및 타겟 사용자

### 1.1 제품 목표

프롬프트 빌더의 1차 목표는 사용자의 모호한 요청을 실행 가능한 AI 프롬프트로 변환하는 것이다. 이를 위해 서비스는 단순한 텍스트 변환기가 아니라, 사용자의 의도 파악, 목적 분류, 템플릿 매칭, 누락 정보 보완, 결과물 미리보기 및 복사까지 하나의 흐름으로 제공해야 한다.

### 1.2 비즈니스 목표

- AI 입문자의 초기 사용 장벽을 낮춰 반복 사용을 유도한다.
- 직장인의 업무 생산성 향상 사례를 확보해 B2C 및 B2B 확장 기반을 만든다.
- 프롬프트 템플릿과 사용자 입력 데이터를 기반으로 향후 유료 템플릿, 팀 공유, 업무별 프롬프트 라이브러리 기능으로 확장한다.

### 1.3 핵심 타겟 사용자

| 사용자군 | 주요 니즈 | 대표 상황 | 핵심 제공 가치 |
| --- | --- | --- | --- |
| AI 입문 일반인 | AI에게 무엇을 어떻게 물어봐야 할지 모름 | 여행 계획, 자기소개서, 학습 계획, 생활 정보 요청 | 질문형 입력을 통해 자연스럽게 완성형 프롬프트 생성 |
| 업무 생산성 향상 직장인 | 반복 업무를 빠르게 처리하고 싶음 | 보고서 초안, 이메일 작성, 회의록 정리, 기획안 작성 | 업무 유형별 템플릿과 결과 형식 지정으로 즉시 활용 가능 |
| 소규모 사업자 및 프리랜서 | 마케팅, 고객 응대, 콘텐츠 제작 자동화 필요 | 홍보 문구, SNS 콘텐츠, 제안서, 고객 안내문 | 목적 기반 프롬프트 생성으로 외부 전문가 없이 작업 지원 |

### 2.2 입력값 처리 로직

서비스는 사용자의 입력을 다음 필드로 구조화해 내부 데이터 모델에 저장한다.

| 필드명 | 설명 | 필수 여부 | 예시 |
| --- | --- | --- | --- |
| goal | 사용자가 AI로 달성하려는 목적 | 필수 | 신제품 소개 이메일 작성 |
| audience | 결과물을 읽거나 사용할 대상 | 권장 | 기존 고객, 팀장, 초등학생 |
| domain | 작업 분야 또는 맥락 | 권장 | 마케팅, 개발, 교육, 인사 |
| outputType | 원하는 결과물 형식 | 필수 | 이메일, 보고서, 표, 체크리스트 |
| tone | 문체 및 분위기 | 선택 | 전문적, 친근한, 설득력 있는 |
| constraints | 제약조건 | 선택 | 500자 이내, 표 형식, 쉬운 단어 사용 |
| sourceText | 참고 자료 또는 원문 | 조건부 | 회의록, 기사, 메모 |
| language | 출력 언어 | 선택 | 한국어, 영어 |

입력 처리 규칙은 다음과 같다.

- goal 또는 outputType이 비어 있으면 최종 프롬프트 생성 버튼을 비활성화한다.
- audience가 비어 있으면 "누가 이 결과물을 보나요?" 질문을 추가로 노출한다.
- outputType이 "요약" 또는 "분석"인 경우 sourceText 입력을 권장 필드로 표시한다.
- tone이 비어 있으면 기본값을 "명확하고 실용적인"으로 설정한다.
- language가 비어 있으면 사용자의 UI 언어를 기본 출력 언어로 사용한다.
- constraints는 배열 형태로 저장하며, 사용자가 자유 입력한 문장을 쉼표 또는 줄바꿈 기준으로 분리한다.
- 사용자가 템플릿을 직접 선택하지 않으면 goal과 outputType 키워드를 기준으로 추천 템플릿을 자동 매칭한다.

### 2.3 프롬프트 조립 로직

최종 프롬프트는 아래 구조를 기본값으로 생성한다.

```text
Role:
당신은 [domain] 분야에 능숙한 전문가입니다.

Context:
사용자는 [goal]을 달성하려고 합니다. 결과물의 대상은 [audience]입니다.

Task:
[outputType] 형식으로 결과물을 작성하십시오.

Constraints:
- 문체는 [tone]을 유지하십시오.
- 다음 제약조건을 준수하십시오: [constraints]

Reference:
[sourceText]

Output Format:
[outputType]에 적합한 구조로 작성하십시오.
```

템플릿별로 일부 섹션은 다르게 조립한다.

- 이메일 템플릿: 제목, 인사말, 본문, CTA, 마무리 인사를 포함한다.
- 보고서 템플릿: 배경, 핵심 요약, 상세 분석, 제안, 다음 액션을 포함한다.
- 마케팅 문구 템플릿: 타겟 고객, 핵심 메시지, 혜택, 행동 유도 문구를 포함한다.
- 개발 요청 템플릿: 기술 환경, 구현 목표, 입력값, 출력값, 예외 처리, 테스트 기준을 포함한다.
- 학습 계획 템플릿: 학습 목표, 현재 수준, 기간, 단계별 커리큘럼, 점검 방법을 포함한다.

### 2.4 템플릿 추천 로직

템플릿 추천은 초기 버전에서는 규칙 기반으로 구현한다.

| 조건 | 추천 템플릿 |
| --- | --- |
| goal에 "메일", "이메일", "답장" 포함 | 이메일 작성 템플릿 |
| goal에 "요약", "정리", "회의록" 포함 | 요약 및 정리 템플릿 |
| goal에 "기획", "제안", "전략" 포함 | 기획서 작성 템플릿 |
| goal에 "광고", "SNS", "홍보", "마케팅" 포함 | 마케팅 카피 템플릿 |
| goal에 "코드", "개발", "버그", "API" 포함 | 개발 요청 템플릿 |
| goal에 "공부", "학습", "강의", "커리큘럼" 포함 | 학습 계획 템플릿 |

향후 버전에서는 사용자 입력을 임베딩하거나 LLM 분류 API를 활용해 템플릿 추천 정확도를 개선한다.

## 3. 사용자 여정 (User Journey)

### 3.1 핵심 사용자 흐름

| 단계 | 사용자 행동 | 시스템 반응 | 성공 조건 |
| --- | --- | --- | --- |
| 1 | 사용자가 서비스에 접속 | 최근 사용 템플릿 또는 시작 화면 표시 | 3초 이내 초기 화면 로딩 |
| 2 | "무엇을 만들고 싶나요?" 질문에 답변 | goal 필드 저장 및 관련 템플릿 추천 | 사용자가 별도 학습 없이 목적 입력 가능 |
| 3 | 추천 템플릿을 선택하거나 직접 선택 | 선택된 템플릿에 맞는 후속 질문 표시 | 템플릿 선택률 70% 이상 |
| 4 | 대상, 결과 형식, 톤, 제약조건 입력 | 입력값을 구조화하고 누락 정보 감지 | 필수 필드 누락 시 명확한 안내 제공 |
| 5 | 사용자가 미리보기 확인 | 실시간으로 최종 프롬프트 조립 | 사용자가 수정할 지점을 쉽게 파악 |
| 6 | 필요 시 답변 수정 | 변경 사항을 프롬프트에 즉시 반영 | 새로고침 없이 반영 |
| 7 | 복사 버튼 클릭 | 클립보드 복사 및 성공 메시지 표시 | 복사 성공률 98% 이상 |
| 8 | 외부 AI 도구에 붙여넣기 | 사용자는 완성된 프롬프트 활용 | 생성 완료 후 이탈이 긍정적 행동으로 간주 |

### 3.2 상세 화면 흐름

1. 시작 화면
   - 핵심 질문 입력창: "AI로 어떤 결과물을 만들고 싶나요?"
   - 추천 템플릿 카드: 이메일, 보고서, 요약, 마케팅, 개발, 학습
   - 최근 사용 템플릿 영역

2. 질문 화면
   - 단계형 질문 UI 제공
   - 모바일에서는 한 화면에 하나의 질문을 집중 표시
   - 데스크톱에서는 왼쪽 질문 패널, 오른쪽 미리보기 패널 구성

3. 미리보기 화면
   - 최종 프롬프트를 섹션 단위로 표시
   - 부족한 항목은 경고 배지로 표시
   - 사용자가 각 섹션을 직접 수정할 수 있도록 편집 모드 제공

4. 완료 화면
   - 복사 버튼을 주요 CTA로 배치
   - "다른 템플릿으로 다시 만들기", "새 프롬프트 만들기" 보조 액션 제공
   - 로그인 사용자는 히스토리에 자동 저장

### 3.3 예외 흐름

| 상황 | 처리 방식 |
| --- | --- |
| 사용자가 너무 짧게 입력 | 예시를 포함한 보완 질문 표시 |
| 필수값 누락 | 생성 버튼 비활성화 및 누락 필드 안내 |

| 모바일 키보드로 화면이 가려짐 | 입력 영역 자동 스크롤 및 하단 CTA 고정 |

## 4. 기술적 요구사항 및 제약 사항

### 4.1 권장 기술 스택

| 영역 | 권장 스택 | 선정 이유 |

| AI 연동 | OpenAI API 또는 호환 LLM API | 향후 프롬프트 개선, 자동 분류, 품질 점수 고도화 |

### 4.3 반응형 UI 요구사항

- 모바일: 360px 이상 화면에서 핵심 플로우 사용 가능해야 한다.
- 태블릿: 질문 영역과 미리보기 영역을 상하 또는 2단 레이아웃으로 제공한다.
- 데스크톱: 왼쪽 입력 패널, 오른쪽 실시간 미리보기 패널의 2컬럼 레이아웃을 기본으로 한다.
- 하단 CTA는 모바일에서 고정 배치하되, 키보드 활성화 시 입력 영역을 가리지 않아야 한다.
- 템플릿 카드는 모바일에서 가로 스크롤 또는 2열 그리드로 표시한다.
- 복사 버튼과 생성 버튼은 터치 영역 최소 44px 이상을 확보한다.

### 4.5 MVP 범위

MVP에 포함할 기능은 다음과 같다.

- 템플릿 6종 제공: 이메일, 요약, 보고서, 마케팅, 개발 요청, 학습 계획
- 인터랙티브 질문 기반 입력
- 규칙 기반 템플릿 추천
- 최종 프롬프트 미리보기
- 클립보드 복사
- 반응형 웹 UI
- 최근 생성 프롬프트 로컬 저장

MVP에서 제외할 기능은 다음과 같다.

- 팀 협업 기능
- 유료 결제
- 고급 AI 기반 자동 개선
- 조직 단위 템플릿 관리
- 브라우저 확장 프로그램

## 5. 성공 지표 (KPI)

### 5.1 핵심 KPI

| KPI | 정의 | 목표 |
| --- | --- | --- |
| 프롬프트 생성 완료율 | 시작 화면 진입 후 최종 프롬프트 생성까지 완료한 사용자 비율 | MVP 출시 후 4주 내 60% 이상 |
| 복사 전환율 | 생성된 프롬프트를 복사한 사용자 비율 | 50% 이상 |
| 평균 생성 소요 시간 | 첫 입력부터 복사까지 걸린 평균 시간 | 3분 이내 |
| 템플릿 선택률 | 직접 또는 추천 템플릿을 선택한 세션 비율 | 70% 이상 |
| 재방문율 | 7일 내 재방문 사용자 비율 | 25% 이상 |
| 사용자 만족도 | 생성 결과에 대한 5점 척도 평가 | 평균 4.0 이상 |

### 5.2 보조 KPI

| KPI | 정의 | 활용 목적 |
| --- | --- | --- |

| 템플릿별 사용 비중 | 템플릿 카테고리별 생성 수 | 우선 개선 템플릿 선정 |

## 출시 기준

다음 조건이 충족되면 MVP 출시 가능 상태로 판단한다.

- 핵심 템플릿 6종에서 정상적으로 프롬프트가 생성된다.
- 필수 입력값 누락 시 생성 버튼이 비활성화되고 보완 안내가 표시된다.
- 모바일 360px, 태블릿 768px, 데스크톱 1440px 기준에서 주요 UI가 깨지지 않는다.
- 복사 기능이 주요 브라우저에서 정상 동작한다.
- 이벤트 트래킹이 핵심 퍼널 기준으로 수집된다.
- 프롬프트 생성 완료율, 복사 전환율, 단계별 이탈률을 대시보드에서 확인할 수 있다.
